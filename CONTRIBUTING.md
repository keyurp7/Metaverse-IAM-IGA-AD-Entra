# Contributing to Metaverse IAM Lab

First off, thank you for taking the time to look at this project. Contributions, feedback, and suggestions are always welcome.

## How You Can Contribute

### Report an Issue
If you spot a broken link, inaccurate documentation, or something that could be improved, please [open an issue](../../issues/new/choose). Use the appropriate template and include as much detail as you can.

### Suggest an Improvement
Have an idea for a new lab scenario, an additional integration, or a better way to document something? Open a feature request issue and describe your suggestion.

### Submit a Pull Request
1. **Fork** the repository
2. **Create a branch** from `main` (`git checkout -b feature/your-improvement`)
3. **Make your changes** — keep commits focused and well-described
4. **Test your changes** — make sure all links resolve and markdown renders correctly
5. **Open a Pull Request** against `main` with a clear description of what you've done

### Documentation Fixes
Typos, grammar, and clarity improvements are always appreciated. These can go straight into a PR — no issue needed.

## Style Guide

- Use clear, concise language
- Follow the existing heading structure (`##` for sections, `###` for subsections)
- Keep screenshot alt text descriptive
- Use relative links for internal documentation references

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this standard.

## Questions?

If you're unsure about anything, feel free to [open a discussion](../../discussions) or reach out via the channels listed in [SUPPORT.md](SUPPORT.md).
