# Lessons Learned

## Key Lessons

1. IAM depends on both governance logic and infrastructure reliability.
2. Source identity quality affects lifecycle automation outcomes.
3. Connector troubleshooting is a core IAM engineering skill.
4. Hybrid identity needs both sync configuration and health monitoring.
5. Runtime components such as Tomcat, IQService, and connector services matter as much as workflow design.
6. The strongest proof comes from end-to-end validation in the target system.

## Project Value

This lab demonstrates architecture thinking, governance engineering, platform support, troubleshooting, and practical automation design in one working model.
